## 第一章 绪论

* **基本概念:**
  * **数据 (Data):** 描述事物的符号记录，可以是数值、文字、图像等。
  * **数据库 (Database, DB):** 长期存储在计算机内，有组织、可共享的数据集合。
  * **数据库管理系统 (Database Management System, DBMS):** 用于管理数据库的软件系统，负责数据的定义、存取、维护和管理。
  * **数据库系统 (Database System, DBS):** 由数据库、DBMS、应用系统、数据库管理员和用户组成。
* **三种主要数据库模型:**
  * **层次模型:** 用树状结构表示实体之间的关系。
  * **网状模型:** 用图状结构表示实体之间的关系。
  * **关系模型:** 用二维表格（关系）表示实体及实体之间的关系，是目前最主流的模型。
* **数据库系统三级模式和二级映像:**
  * **三级模式:**
    * **概念模式 (Conceptual Schema):** 对全局数据结构的描述，独立于具体的硬件和软件。
    * **外模式 (External Schema):** 针对特定用户的局部数据视图，也称子模式。
    * **内模式 (Internal Schema):** 描述数据在物理存储介质上的表示方式。
  * **二级映像:**
    * **外模式/概念模式映像:** 保证数据的逻辑独立性，即当概念模式发生改变时，外模式不会受到影响。
    * **概念模式/内模式映像:** 保证数据的物理独立性，即当存储结构发生改变时，概念模式不会受到影响。

## 第二章 关系数据库

* **关系数据库系统的基本特征:**
  * 以关系模型为基础，用表格表示数据。
  * 数据之间存在联系，通过键来关联不同表。
  * 具有数据独立性，逻辑独立性和物理独立性。
  * 提供数据查询、更新、管理功能。
* **关系的完整性约束:**
  * **实体完整性:** 主码不能为空。
  * **参照完整性:** 外码值必须是参照表中已存在的主码值或为空。
  * **用户自定义完整性:** 由用户定义的约束，如数据类型、取值范围等。
* **关系代数运算:**
  * **基本运算:** 选择 (σ)、投影 (π)、并 (∪)、差 (-)、笛卡尔积 (×)。
  * **组合运算:** 交 (∩)、连接 (⋈)、除 (÷)。

## 第三章 SQL

* **SQL 的主要功能:**
  * **数据定义 (DDL):** 创建、修改、删除数据库对象，如表、索引、视图等。
    * `CREATE TABLE`, `ALTER TABLE`, `DROP TABLE`, `CREATE INDEX`, `CREATE VIEW`
  * **数据操纵 (DML):** 查询、插入、修改、删除数据。
    * `SELECT`, `INSERT`, `UPDATE`, `DELETE`
  * **数据控制 (DCL):** 控制用户的访问权限。
    * `GRANT`, `REVOKE`
* **基本表的定义、删除与修改:**
  * `CREATE TABLE table_name (column_name datatype [constraints], ...);`
  * `ALTER TABLE table_name ADD/MODIFY/DROP COLUMN column_name;`
  * `DROP TABLE table_name;`
* **索引的概念和特点:**
  * **概念:** 用于加速数据检索的特殊数据结构。
  * **特点:**
    * 提高查询效率，但会降低数据更新速度。
    * 需要额外的存储空间。
  * `CREATE INDEX index_name ON table_name (column_name);`
* **插入、删除、修改数据:**
  * `INSERT INTO table_name (column1, column2, ...) VALUES (value1, value2, ...);`
  * `DELETE FROM table_name WHERE condition;`
  * `UPDATE table_name SET column1 = value1, column2 = value2, ... WHERE condition;`
* **视图:**
  * **概念:** 基于表或其他视图的虚拟表，不实际存储数据，只存储视图定义。
  * `CREATE VIEW view_name AS SELECT ... FROM ... WHERE ...;`

## 第四章 数据库的安全性

* **什么是数据库的安全性:**
  * 保护数据库免受未经授权的访问、修改、破坏，以及确保数据的保密性、完整性和可用性。
* **实现数据库系统安全性的技术和方法:**
  * **用户身份验证:** 用户名/密码、指纹识别等。
  * **访问控制:** 定义用户的权限，如读、写、执行等。
  * **数据加密:** 将数据转换为密文进行存储和传输。
  * **审计:** 记录用户操作，以便追踪和分析安全事件。

## 第五章 数据库完整性

* **什么是数据库的完整性:**
  * 确保数据库中数据的正确性、一致性和有效性。
* **关系数据库管理系统完整性实现的机制:**
  * **约束机制:** 包括实体完整性约束、参照完整性约束、用户自定义完整性约束。
  * **触发器:** 定义特定事件（如插入、删除、更新）发生时自动执行的操作。
  * **断言:** 定义必须满足的条件。

## 第六章 关系数据理论

* **函数依赖的基本概念:**
  * **函数依赖 (Functional Dependency, FD):** 关系模式中属性之间的制约关系，如 X → Y，表示在关系中，X 的值确定 Y 的值。
* **完全函数依赖、部分函数依赖、传递函数依赖:**
  * **完全函数依赖:** 如果 X 能决定 Y，并且 X 的任何真子集都不能决定 Y。
  * **部分函数依赖:** 如果 X 能决定 Y，且 X 的某个真子集也能决定 Y。
  * **传递函数依赖:** 如果 X 能决定 Y，Y 能决定 Z，且 Y 不能决定 X，则 X 传递决定 Z。
* **候选码、主码:**
  * **候选码 (Candidate Key):** 能唯一标识关系中元组的最小属性集，一个关系可以有多个候选码。
  * **主码 (Primary Key):** 从候选码中选出一个作为关系的主码。
* **1NF、2NF、3NF、BCNF、4NF 的含义:**
  * **1NF (First Normal Form):** 关系中的每个属性都不可再分。
  * **2NF (Second Normal Form):** 满足 1NF，且非主属性完全依赖于主码。
  * **3NF (Third Normal Form):** 满足 2NF，且非主属性不传递依赖于主码。
  * **BCNF (Boyce-Codd Normal Form):** 满足 3NF，且每个决定因素都包含主码。
  * **4NF (Fourth Normal Form):** 满足 BCNF，且不包含多值依赖。

## 第七章 数据库设计

* **数据库的设计过程 (重中之重):**
  1. **需求分析:** 理解用户的需求，明确数据范围、业务规则。
  2. **概念结构设计:** 将需求转换为概念模型，通常使用 E-R 图。
  3. **逻辑结构设计:** 将概念模型转换为特定的逻辑模型（如关系模型）。
  4. **物理结构设计:** 确定数据的物理存储结构，如索引、存储方式等。
  5. **数据库实施:** 根据设计创建数据库，导入数据。
  6. **数据库运行与维护:** 监控数据库运行状态，定期维护。

## 第八章 数据库编程

* **存储过程的概念和特点:**
  * **概念:** 存储在数据库服务器中的预编译的 SQL 代码块，可以包含控制语句、循环、变量等。
  * **特点:**
    * 提高数据库访问性能。
    * 减少网络流量。
    * 可以模块化地组织业务逻辑。
    * 增强数据库安全性。

## 第十章 关系查询处理和查询优化

* **代数优化和物理优化的概念:**
  * **代数优化:** 对关系代数表达式进行等价变换，减少中间结果的数量。
  * **物理优化:** 选择合适的物理存取路径，如使用索引、选择连接方式等。

## 第十一章 数据库恢复技术

* **事务的概念和性质:**
  * **事务 (Transaction):** 数据库操作的基本逻辑单元，要么全部执行，要么全部不执行。
  * **ACID 特性:**
    * **原子性 (Atomicity):** 事务是不可分割的，要么全部执行成功，要么全部失败回滚。
    * **一致性 (Consistency):** 事务执行前后，数据库从一个一致性状态转换到另一个一致性状态。
    * **隔离性 (Isolation):** 并发事务之间相互隔离，不受影响。
    * **持久性 (Durability):** 已提交的事务对数据库的修改是永久性的。
