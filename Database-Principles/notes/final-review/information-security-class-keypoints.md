# 数据库设计重点内容讲解总结

## 数据库设计重点内容讲解
### 总体情况
- 考试题型多样（选择、填空、简答、综合），共约31题，覆盖面广，难度中等偏下。
- 重点考察基本概念和基础操作，不涉及复杂函数或高级系统功能。
- 强调动手实践，避免只听课不练习。

## 第三章：数据定义与操作
1. **表的定义、删除、修改**
   - 注意区分**表结构**的删除/修改（DDL）与**表中数据**的删除/修改（DML）。
   - 关键词不同，需掌握基本语句。
2. **索引**
   - 掌握索引的创建。
3. **数据查询**
   - 单表查询：包括条件查询（单条件/多条件）、分组（GROUP BY、HAVING）。
   - 连接查询：涉及两个表，类似于书上第三章的例子。
   - 不涉及：子查询中的分组、存在量词（EXISTS）、日期函数、字符串函数等复杂内容。
4. **数据插入、删除、修改**
   - 区分表操作与数据操作的关键词。
5. **视图**
   - 掌握视图的定义（CREATE VIEW AS）。
   - 理解视图的作用。

## 第四章：数据安全性与完整性
1. **安全性**
   - 理解数据库安全性的概念。
   - 掌握存取控制：
     - 自主存取控制（GRANT授权、REVOKE回收）。
     - 强制存取控制（了解即可）。
   - 了解其他安全措施：审计、登录控制等（知道有哪些手段即可，不考实现细节）。
2. **完整性**
   - 掌握三类完整性：实体完整性、参照完整性、用户定义的完整性。
   - 理解并能在定义中表达完整性约束（如外键、级联删除/更新）。

## 第五章：触发器
- 理解触发器的作用、好处。
- 不要求编写触发器定义，只考概念（选择/填空）。

## 第六章：关系数据理论（重点+难点）
- 仅考察**函数依赖**范畴，不涉及多值依赖与第四范式。
1. **函数依赖**
   - 理解函数依赖、部分函数依赖、传递函数依赖。
2. **范式判断**
   - 掌握候选码的求解。
   - 能判断关系模式属于第几范式（1NF、2NF、3NF、BCNF）：
     - 步骤：找候选码 → 判断是否存在部分依赖（若无则为2NF）→ 判断是否存在传递依赖（若无则为3NF）。
   - 了解如何通过分解提升范式级别。
3. **Armstrong公理与闭包**
   - 理解公理，不要求背诵。
   - 会求函数依赖集的闭包。

## 第七章：数据库设计（重点）
1. **设计六个阶段**
   - 要求记住：需求分析 → 概念结构设计 → 逻辑结构设计 → 物理结构设计 → 数据库实施 → 数据库运行与维护。
2. **重点任务**
   - **需求分析**：主要任务（能用自己的话说明）。
   - **E-R图设计与转换**：
     - 能根据语义绘制E-R图（分清1:1、1:n、m:n关系）。
     - 能将E-R图转换为关系模式，并正确设置主码。
   - **三级模式的形成阶段**：
     - 模式：逻辑结构设计阶段。
     - 外模式：逻辑结构设计阶段。
     - 内模式：物理结构设计阶段（涉及存储结构、索引等）。
   - **物理结构设计**：了解索引设计、聚簇索引等。
   - **数据库运行与维护**：了解DBA的主要职责和工作内容。

## 第八章：数据库编程
- 考察形式：选择/填空。
1. **过程化SQL**
   - 了解其背景与构成。
2. **游标（Cursor）**
   - 理解游标的作用（在面向集合的SQL与面向记录的高级语言之间建立桥梁）。
3. **存储过程（Procedure）**
   - 理解其定义、特点、作用。
   - 与触发器的区别：存储过程需显式调用，触发器是自动触发执行。

## 第九章：存储管理
- 考察形式：选择/填空。
- 了解索引的作用与好处。
- 了解常见索引类型：B+树索引、哈希索引（知道概念即可）。
- 不要求记录存储细节（定长/变长记录等）。

## 第十章：查询优化
- 理解两类优化：
  1. **代数优化**：基于关系代数表达式的等价变换（如选择运算尽早执行）。
  2. **物理优化**：基于存取路径的优化（如创建索引）。
- 不要求记忆具体优化规则，重在理解优化思路与角度。

## 第十一章：恢复技术
- 考察形式：选择/填空。
1. **事务（Transaction）**
   - 理解事务的概念。
   - 掌握事务的四个特性（ACID）：原子性、一致性、隔离性、持久性。
2. **故障与恢复**
   - 了解故障类型。
   - 掌握两种主要恢复技术：**日志（Log）** 与**数据转储**。
   - 理解检查点（Checkpoint）的作用：缩短恢复时间，减少不必要的REDO/UNDO。
   - 能区分REDO（重做）与UNDO（撤销）的适用场景。

## 第十二章：并发控制
- 考察形式：选择/填空。
1. **并发问题**
   - 了解可能引发的不一致问题：丢失修改、不可重复读、读脏数据等。
2. **封锁技术**
   - 理解共享锁（S锁）与排他锁（X锁）的作用与区别。
   - 了解三级封锁协议（知道有区别即可，不要求背细节）。
3. **死锁与活锁**
   - 理解概念，能区分。

## 复习建议
- **全面覆盖**：ppt所列内容均可能考察，需全面复习。
- **理解为主**：概念不要求死记硬背，但需理解其含义与区别。
- **重视基础**：SQL语句、范式判断、E-R图转化等基础操作务必动手练习。
- **把握整体**：对于安全性、优化、恢复等内容，重在理解实现思路与可用手段，而非具体技术细节。
