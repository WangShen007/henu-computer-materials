1. **结果如下：**

2. **参照完整性规则**：
   若属性（或属性组）F 是基本关系 R 的外码，它与基本关系 S 的主码 Ks 相对应（基本关系 R 和 S 不一定是不同的关系），则对于 R 中每个元组在 F 上的值必须为：

- 或者取空值（F 的每个属性值均为空值）；
- 或者等于 S 中某个元组的主码值。
  **举例**：在学生关系 (学号, 姓名, 专业号) 和 专业关系 (专业号, 专业名) 中，学生关系中的“专业号”是外码，参照专业关系的主码“专业号”。学生表中的专业号必须是专业表中存在的专业号，或者是空值（表示尚未分配专业）。

3. **数据库设计过程**：
   (1) **需求分析**：了解用户需求，形成需求说明书。
   (2) **概念结构设计**：将需求转化为概念模型（如 E-R 图）。
   (3) **逻辑结构设计**：将概念模型转化为特定的数据模型（如关系模型），并进行优化。
   (4) **物理结构设计**：为逻辑数据模型选取最适合应用环境的物理结构（存储结构和存取方法）。
   (5) **数据库实施**：建立数据库，编制与调试应用程序，组织数据入库，进行试运行。
   (6) **数据库运行和维护**：从数据库试运行开始，对数据库进行评价、调整、修改。
4. **存储过程的优点**：
   (1) **运行速度快**：存储过程在服务器端预编译和优化，执行效率高。
   (2) **减少网络流量**：客户端只需要发送调用语句，不需要发送复杂的 SQL 代码。
   (3) **增强安全性**：可以通过权限设置限制用户对自己存储过程的访问，而不需要直接授权访问底层表。
   (4) **模块化编程**：一次编写，多次调用，易于维护和修改。
5. **答：**
   (1) **基本函数依赖**：

- 学生姓名 $\rightarrow$ 宿舍
- 课程名 $\rightarrow$ 学分
- (学生姓名, 课程名) $\rightarrow$ 成绩
  (2) **候选码**：
- (学生姓名, 课程名)
  *注：假设“学生姓名”能唯一标识学生，“课程名”能唯一标识课程。*

6. **答：**
   **事务定义**：事务是用户定义的一个数据库操作序列，这些操作要么全做，要么全不做，是一个不可分割的工作单位。
   **事务的 4 个特性 (ACID)**：
   (1) **原子性 (Atomicity)**：事务是数据库的逻辑工作单位，事务中包括的诸操作要么都做，要么都不做。
   (2) **一致性 (Consistency)**：事务执行的结果必须是使数据库从一个一致性状态变到另一个一致性状态。
   (3) **隔离性 (Isolation)**：一个事务的执行不能被其他事务干扰。
   (4) **持续性 (Durability)**：一个事务一旦提交，它对数据库中数据的改变就应该是永久性的。

## 四、 编程题

1. **查询参加了G3小组的学生姓名和获奖情况**

```sql
SELECT S.SNAME, J.PRIZE
FROM S, J
WHERE S.SNO = J.SNO AND J.GNO = 'G3';
```

2. **查询参加了“我爱大自然”小组的学生学号和姓名**

```sql
SELECT S.SNO, S.SNAME
FROM S, J, G
WHERE S.SNO = J.SNO AND J.GNO = G.GNO AND G.GNAME = '我爱大自然';
```

3. **将G4小组名称改为“计算机爱好者”**

```sql
UPDATE G
SET GNAME = '计算机爱好者'
WHERE GNO = 'G4';
```

4. **插入记录**

```sql
INSERT INTO S (SNO, SNAME, SAGE, SSEX)
VALUES ('S3', '张三', 20, '男');
```

5. **删除没有参加兴趣小组的学生信息**

```sql
DELETE FROM S
WHERE SNO NOT IN (SELECT DISTINCT SNO FROM J);
```
