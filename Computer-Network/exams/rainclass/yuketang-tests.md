# 计网雨课堂第一次课测试

## 一、OSI七层模型与TCP/IP模型

### OSI七层模型

| 层次 | 名称 | 功能 | 协议/设备 |
|-----|------|------|----------|
| 7 | 应用层（Application） | 为应用程序提供网络服务 | HTTP, FTP, SMTP, DNS |
| 6 | 表示层（Presentation） | 数据格式转换、加密解密 | SSL/TLS, JPEG, ASCII |
| 5 | 会话层（Session） | 建立、管理和终止会话 | RPC, NetBIOS |
| 4 | 传输层（Transport） | 端到端可靠传输 | TCP, UDP |
| 3 | 网络层（Network） | 路由和寻址 | IP, ICMP, 路由器 |
| 2 | 数据链路层（Data Link） | 帧传输和差错控制 | Ethernet, 交换机 |
| 1 | 物理层（Physical） | 比特流传输 | 双绞线, 光纤, 集线器 |

### TCP/IP四层模型

| 层次 | 名称 | 对应OSI层 |
|-----|------|----------|
| 4 | 应用层 | 应用层+表示层+会话层 |
| 3 | 传输层 | 传输层 |
| 2 | 网际层（Internet） | 网络层 |
| 1 | 网络接口层 | 数据链路层+物理层 |

## 二、网络分类

### 按覆盖范围分类
- **PAN**（个人区域网）：< 10m，如蓝牙
- **LAN**（局域网）：< 1km，如校园网
- **MAN**（城域网）：1-100km，如城市网络
- **WAN**（广域网）：> 100km，如互联网

### 按拓扑结构分类
- **星型拓扑**：所有节点连接到中心节点（如交换机）
- **总线型拓扑**：所有节点共享一条通信线路
- **环型拓扑**：节点首尾相连形成环
- **网状拓扑**：节点之间有多条路径

## 三、基本概念

### 带宽（Bandwidth）
- 定义：网络通信线路传输数据的能力
- 单位：bps（bits per second）
- 常见单位：Kbps、Mbps、Gbps
- 注意：带宽≠实际传输速率

### 时延（Latency/Delay）
- **发送时延**：数据从发送端发出所需时间 = 数据长度/发送速率
- **传播时延**：信号在信道中传播所需时间 = 距离/传播速度
- **处理时延**：路由器/交换机处理数据包所需时间
- **排队时延**：数据包在缓冲区排队等待的时间

### 吞吐量（Throughput）
- 定义：单位时间内成功传输的数据量
- 受带宽、时延、丢包率等因素影响
- 实际吞吐量通常小于带宽

## 四、网络拓扑结构详解

### 星型拓扑
```
    [PC1]    [PC2]
       \     /
        [SW]
       /     \
    [PC3]    [PC4]
```
- 优点：易于安装和维护，故障隔离好
- 缺点：中心节点故障影响全网

### 总线型拓扑
```
[PC1]---[PC2]---[PC3]---[PC4]
  |___________________________|
            总线
```
- 优点：结构简单，成本低
- 缺点：总线故障影响全网，不易维护

### 环型拓扑
```
[PC1] → [PC2]
  ↑       ↓
[PC4] ← [PC3]
```
- 优点：令牌传递，无冲突
- 缺点：单点故障可能影响全网

### 网状拓扑
```
[PC1]---[PC2]
  |  \ /  |
  |   X   |
  |  / \  |
[PC3]---[PC4]
```
- 优点：高可靠性，多路径冗余
- 缺点：成本高，管理复杂
