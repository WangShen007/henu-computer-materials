课后探研题 3: 网际校验和算法

## 1.1 目的
1. 理解网际校验和算法(Internet CheckSum)的工作原理；
2. 培养协议分析、设计和实现的能力。

## 1.2 内容
自主学习网际校验和算法相关知识，编程实现网际校验和算法。

## 1.3 步骤
1. 收集网际校验和算法的相关资料；
2. 阅读 RFC1071 文档，理解网际校验和算法的原理；
3. 编程实现网际校验和算法；
4. 完成书面报告。

## 1.4 编程要求
### 1.4.1 编程语言
编程语言不限。

### 1.4.2 编程算法
1. 编程实验网际校验和算法；
2. 从实验 5 中捕获报文，报文包括 IP、ICMP、UDP、TCP 等类型，以文件形式保存；
3. 从测试文件读取报文，文件中的报文存储格式不限，自定义；
4. 根据报文类型的不同，调用网际校验和算法进行差错检验；
5. 输出报文内容和差错检验结果；
6. 代码要有详细注释。

## 1.5 书面报告要求
书面报告的正文部分，字数不少于 1500 字。书面报告只需要提交电子文档。

### 1.5.1 书面报告内容要求
书面报告中，至少包括以下内容：
1. 编程环境（操作系统、开发工具、编程语言等）；
2. 报文数据类型定义；
3. 算法说明及流程图、核心代码；
4. 测试用报文文件格式说明；
5. 程序运行结果抓图；
6. 参考文献。

### 1.5.2 参考文献格式要求
参考文献书写格式应符合 GB7714-1987《文后参考文献著录规则》。

### 1.5.3 书面报告格式要求
书面报告的格式要求，见文档“《计算机网络》课后探研题书面报告模版”。

### 1.5.4 书面报告文档命名规范
书面报告文档 PDF 格式，文档命名为：
班级_学号_姓名_题目
如：
17-2_1710120000_张三_了解网络设备

班级_学号_姓名
