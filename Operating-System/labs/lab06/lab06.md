**课程名称**: 操作系统
**实验名称**: 虚拟存储器

## 一、 实验目的
本实验旨在通过编程模拟，加深对请求分页存储管理系统的理解。具体目标包括：
1.  掌握虚拟存储器中硬件地址变换的模拟方法。
2.  理解并实现页面置换算法（FIFO 和 LRU），掌握其在缺页中断处理中的作用。
3.  通过模拟，分析不同置换算法在给定指令序列下的缺页率和性能差异。

## 二、 实验内容
本实验包含以下三个任务的模拟实现：
1.  **硬件地址变换**: 模拟 MMU 逻辑。若页面在内存（Flag=1），计算物理地址；若不在内存（Flag=0），产生缺页中断。
2.  **FIFO (先进先出) 页面置换**: 当产生缺页且内存满时，淘汰最早进入内存的页面。
3.  **LRU (最近最久未使用) 页面置换**: 当产生缺页且内存满时，淘汰最近最长时间未被访问的页面。

**实验参数**:
- 页面大小: 1KB (1024 字节)
- 分配给作业的主存块数: 4 块
- 初始页表状态及指令序列: 依据实验指导书设定。

## 三、 算法原理与数据结构

### 1. 数据结构定义
使用 C++ 结构体定义页表项和指令：

```cpp
// 页表项结构
struct PageItem {
    int pageNo;       // 页号
    int flag;         // 标志: 1-在内存, 0-不在内存
    int blockNo;      // 块号
    string diskAddr;  // 磁盘地址
    int modified;     // 修改位: 1-已修改, 0-未修改
    // 辅助字段
    int enterTime;    // 用于 FIFO
    int lastAccessTime; // 用于 LRU
};
// 指令结构
struct Instruction {
    string op;        // 操作: +, *, 存, 取, 移位等
    int pageNo;       // 页号
    int offset;       // 页内地址
};
```

### 2. 算法逻辑
- **硬件地址变换**:
  - 检查页表项的 `flag`。
  - 若 `flag == 1` (命中): `物理地址 = 块号 * 1024 + 页内偏移`。
  - 若 `flag == 0` (缺页): 触发中断，转入置换算法。

- **FIFO (First-In First-Out, 先进先出)**:
  - 维护一个队列，记录页面进入内存的顺序。
  - 需要置换时，选择队列头部的页面淘汰。
  - 新页面加入队列尾部。

- **LRU (Least Recently Used, 最近最久未使用)**:
  - 维护一个栈（或有序列表），栈顶表示最近访问的页面，栈底表示最久未访问的。
  - **命中时**: 将该页面移至栈顶。
  - **缺页且需置换时**: 淘汰栈底页面。
  - **调入新页**: 将新页面压入栈顶。

## 五、 实验结论与心得
1.  **缺页率相同**: 在本实验设定的特定指令序列长度（12 条）和内存块数（4 块）下，FIFO 和 LRU 算法表现出了相同的缺页率（50%）。
2.  **淘汰策略差异**: 尽管最终统计数据相同，但在具体的置换过程中，两者选择的淘汰页面不同（如指令 6 时，FIFO 淘汰页 0，LRU 淘汰页 1）。这验证了 FIFO 关注“进入时间”，而 LRU 关注“访问历史”的本质区别。
3.  **修改位处理**: 实验中成功模拟了脏页（Modified Bit = 1）的写回操作。当被置换的页面被修改过时，必须将其写回磁盘，这是虚拟存储器保证数据一致性的关键步骤。
4.  **程序设计**: 通过使用 C++ 的 STL 容器（如 `deque` 和 `vector`），方便地模拟了 FIFO 的队列结构和 LRU 的栈/列表结构，简化了从头实现数据结构的复杂度。
